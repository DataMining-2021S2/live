Kaggle Style Competition

Outline

Kaggle is a data science community website that hosts numerous guides, datasets, and runs online competitions. Recently they added the feature of Community Competitions where anybody can host a competition using Kaggle's platform.

Kaggle is a fantastic resource and to encourage you to explore it, this assignment is going to be run using Kaggle's platform.

You'll need a private key to access the competition, which is semi-private. This key can be found in the assignment page on Moodle.

Your task is to analyse the given dataset and construct a suitable classifier. The performance of your classifier is computed based on your uploaded predictions.

Details

  • First create a Kaggle account (using whatever account name/details you wish) and read the competition instructions (Please ask if there is anything that is unclear).
  • Setup a suitable ML project folder.
  • Download the train and test datasets, train.csv and test.csv.
  • Split train.csv into your df_train and df_test datasets.
  • Train a classifier using the workflow covered during the course.
  • Using your model, generate predictions for the data in test.csv and upload.
  • At end of module upload your best scoring set of predictions to Moodle for grading.

Submissions

Slack channel for questions/updates: https://datamining2021s2.slack.com/archives/C02UQU4P50R

Since your Kaggle and Moodle accounts are separate, you will also need to upload to Moodle your predictions for final grading.

You will need to upload two files (uncompressed):

  • A python notebook (Model.ipynb) containing your best preforming model. The notebook should
  • Contain a markdown cell with a list of your submissions containing submission score and outline of the attempt.
  • Should be executable if placed in a folder that has the subfolder data containing the data files downloaded from Kaggle.
  • Your predictions (submission.csv). This should be the file that you uploaded to Kaggle to score.
Submit using Moodle

Grading

The grading for this assignment is based on:

  • The model performance 60%
  • Details of the implemented model and steps attempted to improve to model 40%
