Employees Dataset

Aim

Given details of a sample of employees from Fortune 100 companies in the US, this assignment attempts to answer two (supervised learning) questions:

  1. How likely is it for a given employee to leave their current company in a specified time period? And can we predict this?
  2. How accurately can we predict a given employee's monthly income?

The first of these is a classification problem and the second is a regression problem.

The data includes many categorical and numerical attributes, all of which are candidate features for each model. Training data is provided so that students can derive a model predicting each of these two targets. Test data is also provided, where the target attribute is missing and so needs to be predicted.

The dataset consists of three files:

  • train.csv: List of 1020 employees. The columns can be loosely grouped as

    • personal features consisting of Employee_Number, Age, Gender, Education, Education_Field, Environment_Satisfaction,Marital_Status, Num_Companies_Worked, Over_18, Work_Life_Balance, and Total_Working_Years.
    • company features consisting of Distance_from_Home, Employee_Count, and Years_at_Company.
    • work conditions related features Business_Travel, Department,Job_Involvement, Job_Level, Job_Role, Years_in_Current_Role, Years_since_Last_Promotion, and Years_with_Curr_Manager.
    • job enumeration related features Daily_Rate, Hourly_Rate, Monthly_Rate, Over_Time,Percent_Salary_Hike, Standard_Hours, and Stock_Option_Level.
    • feedback related features Job_Satisfaction, Performance_Rating, Satisfaction_with_Manager, and Training_Times_Last_Year.
    • targets Attrition and Monthly_Income.

  • test_Attrition.csv: List of 200 employees with the Attrition attribute missing, so this target needs to be predicted by a classification model.

  • test_Monthly_Income.csv: List of 250 employees with the Monthly_Income attribute missing, so it needs to be predicted by a regression model.

Note that some of the categorical features have already been coded. Also when predicting Attrition the column Monthly_Income can be treated as a feature, and similarly Attrition can be treated as a feature when predicting Monthly_Income.

Task 0: Setup

  • Create a folder for this assignment. Your notebooks should be saved into this folder.
  • Create subfolder data in your assignment folder and place the three data files (train.csv, test_Attrition.csv, and test_Monthly_Income.csv) into this folder.

Task 1: Exploratory Data Analysis (worth 40% of the marks)

Load data and preform a typical EDA.

  • For each feature generate suitable summary/statistics.
  • Comment on any interesting observations relating to the feature itself or in relation with either target.
  • Suggest which attributes might become valuable features in predictive models.

Remember: Exploratory Data Analysis is important since a successful EDA stage will help to understand the data and to prepare it for further analysis.

Task 2: Predicating Attrition as a target (worth 30% of the marks)

  • Build a classification model to predict the categorical target Attrition.
  • Since the model is imbalanced use the f1 metric to score the model.
  • Use the model to generate predictions for the cases in the data/test_Attrition.csv file.
  • Determine which features are important in predicting Attrition.
  • Comment on the quality of the model, as a predictor of Attrition.
  • Save predictions to CSV file pred_Attrition.csv.

The choice of classification algorithm is left to the student, who should justify this choice based on the characteristics of the data and of the task.

Task 3: Predicating Monthly_Income as a target (worth 30% of the marks)

  • Build a regression model to predict the numerical target Monthly_Income.
  • Use root mean square error (RMSE) to score the model.
  • Use the model to generate predictions for the cases in file data/test_Monthly_Income.csv.
  • Use plots to check that the standard assumptions for regression are met by the model you propose.
  • Comment on the quality of the model, as a predictor of Monthly_Income, and how you would interpret the model.
  • Save predictions to CSV file pred_Monthly_Income.csv.
