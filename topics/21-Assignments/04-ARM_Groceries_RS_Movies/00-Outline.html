Assessment 4: ARM and Recommendation tasks

Aim

The purpose of this assessment is to give students the opportunity to show what they have learned in the module by doing data mining, particularly association analysis and recommender systems.

The dataset for each is well-known (the grocery.csv set for the former, and the smallest of the Movielens movie rating databases for the latter).

Specific tasks are set, relating to how the analysis is set up, how to configure the machine learning techniques, and how to interpret and/or improve the results.

Format of submissions

Students are asked to submit two Jupyter notebooks, one for each part of the assessment, together with any supporting resources needed to run the analysis. If you use any additional python packages, please indicate where they can be found too.

Students are reminded to use the markdown cells in their notebooks to explain their reasoning, discuss outputs and report on the progress of their data investigation. Students are also advised to use both tabular and plotted output to present their work. Each notebook should read like a technical report, with all findings supported by evidence.

Submission mode and Deadline

Assignment attempts should be uploaded via the Moodle submission page. You can upload as often as you wish, but please remember to submit the version you wish us to grade.

The deadline for submission is Sunday 15 May 2022 at 23:00.

Marking scheme

id Criterion Marks
1 ARM: Select transactions for a country and prepare vertical format dataframe 10
2 ARM: Derive association rules whose lift is 1 and process them 15
3 ARM: Compare and recommend one of two rules that differ by direction 15
4 RS: Read data and preprocess it to make it ready 10
5 RS: Apply filter and visualise movie count 10
6 RS: Investigate rating counts and averages 25
7 RS: Benchmark recommender algorithms in terms of their error and their timings 15

This assignment is worth 20% of your overall grade for this module.

Detailed specification

A detailed specification has been prepared in the format of a python notebook for each section.

This includes advice on how to obtain the data and how to perform some initial processing so that the data is ready for the next stage of processing.

Association analysis and recommendation tasks are set and students are asked to investigate the data using these approaches. If a student wishes to add extra forms of analysis which prove enlightening, bonus marks might be added, but the core tasks should be attempted at the very least.

For more details, please refer to the ARM specification and RS specification notebooks.

What to do next

Compared to Assessment 3, the tasks are more explicit and so students should begin by attempting to complete them, using what they have learnt from Week 09 (Association Analysis) and Week 10 (Anomaly Detection and Recommender Systems), supplemented as always by the EDA and model building approaches they have learnt earlier in the module.

There is a slack channel for questions/updates on CA4.
